⏺ Ant Design vs MoinUI Avatar 差异
  特性: Size
  Ant Design: number | large | small | default | { xs, sm, md, lg, xl, xxl } 响应式
  MoinUI: .large | .default | .small | .custom(CGFloat) ❌无响应式
  ────────────────────────────────────────
  特性: Gap
  Ant Design: gap?: number 字符距边界像素，默认4
  MoinUI: ❌ 无
  ────────────────────────────────────────
  特性: Src
  Ant Design: string | ReactNode, 支持 srcSet, crossOrigin, draggable, alt
  MoinUI: 仅 Image 或 @ViewBuilder
  ────────────────────────────────────────
  特性: onError
  Ant Design: () => boolean 图片加载失败回调，可阻止fallback
  MoinUI: ❌ 无
  ────────────────────────────────────────
  特性: Icon
  Ant Design: ReactNode 任意节点
  MoinUI: String 仅 SF Symbol
  ────────────────────────────────────────
  特性: 文字缩放
  Ant Design: ResizeObserver 精确计算 scale
  MoinUI: 简单 minimumScaleFactor(0.5)
  AvatarGroup 差异
  ┌──────────┬────────────────────────────────────┬─────────────────────┐
  │   特性   │             Ant Design             │       MoinUI        │
  ├──────────┼────────────────────────────────────┼─────────────────────┤
  │ Max      │ { count, style, popover } 完整配置 │ 仅 maxCount: Int?   │
  ├──────────┼────────────────────────────────────┼─────────────────────┤
  │ 溢出展示 │ Popover 悬浮显示隐藏头像           │ ❌ 仅显示 +N        │
  ├──────────┼────────────────────────────────────┼─────────────────────┤
  │ RTL      │ ✅ 支持                            │ ❌ 无               │
  ├──────────┼────────────────────────────────────┼─────────────────────┤
  │ Context  │ 通过 Context 向下传递 size/shape   │ ❌ 子元素需自行设置 │
  └──────────┴────────────────────────────────────┴─────────────────────┘
  核心缺失

  1. 响应式尺寸 - { xs: 24, sm: 32, md: 40... } 按断点自适应
  2. 图片错误回调 - onError + fallback 机制
  3. Gap 参数 - 文字边距控制
  4. Group Popover - 溢出头像悬浮展示
  5. Group Context - 统一传递 size/shape 给子元素




  需要支持gap，与 antd api一致
  src支持传递图片网址，而不是现在 image 那样的是个图标，同时图片获取失败有一个兜底图标显示。
  Icon 我的也需要支持任意 Icon: 重载View
  我要坚持精确计算 scale。
  group 也要一样的实现
